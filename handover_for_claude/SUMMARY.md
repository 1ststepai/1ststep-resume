# Handover Summary — Navigation & Layout Standardization

This document provides a high-level summary of the architectural changes made during the April 2026 refactor. The platform has been transitioned from a legacy, content-heavy scrolling page to a modern, app-like SPA with a standardized layout and dedicated workflow modes.

## 1. Architectural Shift: The "App Shell"
The entire application is now wrapped in a `.app` container with a fixed `100vh` height and `overflow: hidden`. 
- **Legacy**: Browsers handled global scrolling of the entire page.
- **Modern**: The **Main Panel** and **Sidebar** handle their own internal scrolling. This prevents the "double scrollbar" issue and keeps the navigation fixed.

## 2. Standardized Navigation (SVG Icons)
- **Top Bar**: High-density navigation for primary tools.
- **Side Bar**: Persistent navigation for secondary/utility tools.
- **Design Language**: All icons have been migrated to official SVG v6 icons for a premium feel. Emojis have been phased out of the primary UI.

## 3. Mode-Switching (Hard Mutual Exclusion)
The `switchMode(mode)` function now enforces **Hard Mutual Exclusion**. When switching to a tool like the **Tracker** or **AI Coach**, all other panels are hidden, and the layout mode is adjusted.

### Layout Modes:
- **Split View**: (Default) Left and Right panels for Resume Tailor and Job Search.
- **Centered View (`mode-centered`)**: Concentrated columns for utility tools (Tracker, AI Coach, LinkedIn Audit, etc.).
